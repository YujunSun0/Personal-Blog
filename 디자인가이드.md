# 디자인 가이드 (Design Guide)

## 1. 디자인 철학

### 1.1 레퍼런스
- **기본 레퍼런스**: [토스 기술 블로그](https://toss.tech/)
- **차별화 포인트**: 개인 블로그의 개성과 따뜻함

### 1.2 핵심 원칙
1. **깔끔함 (Simplicity)**: 불필요한 요소 제거, 집중도 높은 콘텐츠 전달
2. **가독성 (Readability)**: 긴 글을 읽기 편한 타이포그래피와 여백
3. **일관성 (Consistency)**: 통일된 디자인 시스템으로 사용자 경험 향상
4. **개성 (Personality)**: 개인 블로그만의 따뜻하고 친근한 느낌

## 2. 컬러 시스템

### 2.1 메인 컬러
```css
/* Primary (메인 브랜드 컬러) */
--color-primary: #3B82F6;        /* Blue-500 - 링크, 버튼, 강조 */
--color-primary-hover: #2563EB;  /* Blue-600 */
--color-primary-light: #DBEAFE;  /* Blue-100 - 배경 강조 */

/* Secondary (보조 컬러) */
--color-secondary: #10B981;     /* Emerald-500 - 성공, 완료 */
--color-secondary-hover: #059669; /* Emerald-600 */
```

### 2.2 중성 컬러 (Neutral)
```css
/* 텍스트 */
--color-text-primary: #171717;   /* Gray-900 - 본문 텍스트 */
--color-text-secondary: #525252; /* Gray-600 - 보조 텍스트 */
--color-text-tertiary: #A3A3A3;  /* Gray-400 - 비활성 텍스트 */

/* 배경 */
--color-bg-primary: #FFFFFF;     /* White - 메인 배경 */
--color-bg-secondary: #F9FAFB;   /* Gray-50 - 섹션 배경 */
--color-bg-tertiary: #F3F4F6;    /* Gray-100 - 카드 배경 */

/* 테두리 */
--color-border: #E5E7EB;         /* Gray-200 - 기본 테두리 */
--color-border-hover: #D1D5DB;   /* Gray-300 - 호버 테두리 */
```

### 2.3 상태 컬러
```css
/* 성공 */

/* 경고 */

/* 에러 */
```

## 3. 타이포그래피

```css

             'Droid Sans Mono', 'Source Code Pro', monospace;
```

### 3.2 폰트 크기
```css
/* 제목 */
--text-4xl: 2.25rem;   /* 36px - 페이지 제목 */
--text-3xl: 1.875rem;  /* 30px - 섹션 제목 */
--text-2xl: 1.5rem;    /* 24px - 카드 제목 */

```

### 3.3 폰트 굵기
```css
--font-bold: 700;      /* 제목, 강조 */

--font-normal: 400;    /* 기본 본문 */

```

### 3.4 줄 간격 (Line Height)

## 7. 메인 페이지 레이아웃

### 7.1 구조 (카카오페이/토스 스타일 참고)
```
┌─────────────────────────────────────────────────────┐
│           Header (고정)                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────┐  ┌──────────────────────────────┐   │
│  │          │  │  [이미지] 제목                │   │
│  │  태그    │  │  설명...                      │   │
│  │  필터    │  │  날짜                         │   │
│  │  (좌측)  │  ├──────────────────────────────┤   │
│  │          │  │  [이미지] 제목                │   │
│  │          │  │  설명...                      │   │
│  │          │  │  날짜                         │   │
│  │          │  │  ... (리스트 형태)            │   │
│  └──────────┘  └──────────────────────────────┘   │
│                                                     │
├─────────────────────────────────────────────────────┤
│           Footer                                    │
└─────────────────────────────────────────────────────┘
```

### 7.2 반응형 레이아웃
- **데스크톱 (1024px+)**: 좌측 태그 필터 + 우측 글 리스트 (가로형 카드)
- **태블릿 (768px~1023px)**: 상단 태그 필터 + 글 리스트 (가로형 카드)
- **모바일 (<768px)**: 상단 태그 필터 (가로 스크롤) + 글 리스트 (세로형 카드)

## 8. 글 카드 디자인 (카카오페이/토스 스타일 참고)

### 8.1 카드 구성 요소 (가로형 레이아웃)
```
┌─────────────────────────────────────────────────────┐
│  [썸네일]  │  [타입 뱃지] [태그들]                  │
│  (280px)   │  제목 (2줄 제한)                      │
│            │  설명 (2줄 제한)                      │
│            │  2024년 1월 15일                     │
└─────────────────────────────────────────────────────┘
```

### 8.2 썸네일
- **크기**: 280px × 180px (고정)
- **위치**: 좌측 (카카오페이 스타일)
- **스타일**: `object-fit: cover`, 둥근 모서리
- **모바일**: 전체 너비, 높이 200px

### 8.3 텍스트 영역
- **제목**: 1.5rem (24px), 굵게, 2줄 제한
- **설명**: 0.9375rem (15px), 2줄 제한
- **날짜**: 0.875rem (14px), 연월일 형식

### 8.4 호버 효과
- 카드가 우측으로 살짝 이동 (`translateX(4px)`)
- 테두리 색상 변경
- 그림자 추가

## 9. 헤더 디자인

### 9.1 구성 요소
```
┌─────────────────────────────────────────┐
│  로고  │  네비게이션  │  [다크모드] [구독] │
└─────────────────────────────────────────┘
```

### 9.2 스타일
- **높이**: 64px (고정)
- **배경**: 흰색 (다크모드: 어두운 회색)
- **하단 테두리**: 얇은 회색 선
- **스크롤 시**: 그림자 추가 (선택)

## 10. 차별화 포인트

### 10.1 토스 기술 블로그와의 차이점
1. **태그 필터 위치**: 좌측 사이드바 (토스는 상단 카테고리)
2. **글 타입 뱃지**: TECH/TROUBLESHOOTING/LIFE 시각적 구분
3. **일상 기록 포함**: LIFE 타입으로 더 따뜻한 느낌
4. **개인 블로그 특성**: 더 개성 있는 헤더/푸터 디자인

### 10.2 개성 있는 요소
- **따뜻한 색감**: LIFE 타입의 초록색으로 일상의 따뜻함 표현
- **부드러운 애니메이션**: 카드 호버, 태그 선택 등 자연스러운 전환
- **읽기 편한 본문**: 넉넉한 줄 간격과 여백

## 11. 다크모드 (향후 적용)

### 11.1 컬러 매핑
- 밝은 배경 → 어두운 배경
- 어두운 텍스트 → 밝은 텍스트
- Primary 컬러는 유지 (약간 조정 가능)

### 11.2 적용 범위
- 모든 페이지
- 모든 컴포넌트
- 코드 블록 하이라이팅

---

## 12. 접근성 (Accessibility)

### 12.1 색상 대비
- 본문 텍스트: 최소 4.5:1 대비율
- 큰 텍스트: 최소 3:1 대비율

### 12.2 키보드 네비게이션
- 모든 인터랙티브 요소에 포커스 표시
- Tab 순서 논리적 배치

### 12.3 ARIA 레이블
- 아이콘 버튼에 `aria-label` 추가
- 폼 필드에 `aria-describedby` 추가

## 13. 애니메이션

### 13.1 전환 시간
```css
--transition-fast: 0.15s;    /* 빠른 전환 */
--transition-base: 0.2s;     /* 기본 전환 */
--transition-slow: 0.3s;    /* 느린 전환 */
```

### 13.2 이징 함수
```css
--ease-in-out: cubic-bezier(0.4, 0, 0.2, 1);
--ease-out: cubic-bezier(0, 0, 0.2, 1);
```

### 13.3 애니메이션 예시
- 카드 호버: `transform: translateY(-4px)`
- 버튼 클릭: 약간의 스케일 다운
- 페이지 전환: 페이드 인/아웃

## 14. 구현 우선순위

3. 버튼
4. 입력 필드

1. 다크모드
2. 애니메이션
3. 반응형 최적화
4. 접근성 개선
